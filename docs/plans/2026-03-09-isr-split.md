# ISR Split Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace `wait_vbl_done()` polling with `vbl_isr` + `lcd_isr` interrupt handlers, giving a HUD/game scroll split at scanline 15.

**Architecture:** `vbl_isr` and `lcd_isr` are static functions in `main.c`. `vbl_isr` sets `frame_ready` and locks HUD scroll; `lcd_isr` fires at `LYC_REG = 15` to apply `cam_y` for game viewport. `state_playing` no longer calls `move_bkg`.

**Tech Stack:** GBDK 4.x, `gb/gb.h` (`add_VBL`, `add_LCD`, `set_interrupts`, `VBL_IFLAG`, `LCD_IFLAG`), `gb/hardware.h` (`STAT_REG`, `LYC_REG`, `STATF_LYC` = `0b01000000`). All via `#include <gb/gb.h>`.

### Task 4: Implement ISRs and replace wait_vbl_done in main.c

**Files:**
- Modify: `src/main.c`

**Step 1: Add camera.h include and frame_ready flag + ISR functions**

In `src/main.c`, add after the existing includes:

```c
#include "camera.h"
```

Then add before `main()`:

```c
static volatile uint8_t frame_ready = 0;

static void vbl_isr(void) {
    frame_ready = 1;
    move_bkg(0, 0);
}

static void lcd_isr(void) {
    move_bkg(0, (uint8_t)cam_y);
}
```

**Step 2: Register ISRs and configure STAT in main()**

In `main()`, inside the `DISPLAY_OFF` block, after `player_init()` and before `DISPLAY_ON`:

```c
    add_VBL(vbl_isr);
    add_LCD(lcd_isr);
    LYC_REG  = HUD_SCANLINE;
    STAT_REG |= STATF_LYC;
    set_interrupts(VBL_IFLAG | LCD_IFLAG);
```

**Step 3: Replace wait_vbl_done() in the main loop**

Change:
```c
    while (1) {
        wait_vbl_done();
        input_update();
        state_manager_update();
    }
```

To:
```c
    while (1) {
        while (!frame_ready);
        frame_ready = 0;
        input_update();
        state_manager_update();
    }
```

The full resulting `main()` should look like:

```c
void main(void) {
    DISPLAY_OFF;

    init_palettes();
    player_init();
    add_VBL(vbl_isr);
    add_LCD(lcd_isr);
    LYC_REG  = HUD_SCANLINE;
    STAT_REG |= STATF_LYC;
    set_interrupts(VBL_IFLAG | LCD_IFLAG);

    DISPLAY_ON;

    state_manager_init();
    state_push(&state_title);

    while (1) {
        while (!frame_ready);
        frame_ready = 0;
        input_update();
        state_manager_update();
    }
}
```

### Task 5: Remove move_bkg from state_playing.c

`lcd_isr` now owns scroll application. The direct `move_bkg(0, (uint8_t)cam_y)` call in `update()` is redundant and must be removed.

**Files:**
- Modify: `src/state_playing.c`

**Step 1: Delete the move_bkg call**

Remove these lines from `update()` in `src/state_playing.c`:

```c
    /* SCY is 8-bit and wraps at 256. Truncation is correct: stream_row() places
     * world row ty at VRAM row (ty & 31), so (uint8_t)cam_y correctly indexes
     * the ring buffer. */
    move_bkg(0u, (uint8_t)cam_y);
```

The `update()` body should now read:

```c
static void update(void) {
#ifdef DEBUG
    frame_count++;
    if (frame_count % 60u == 0u) {
        DBG_INT("frame", (int)frame_count);
        DBG_INT("px", (int)player_get_x());
        DBG_INT("py", (int)player_get_y());
    }
#endif
    /* VBlank phase: all VRAM writes immediately after frame_ready */
    player_render();
    camera_flush_vram();
    /* Game logic phase: runs during active display */
    player_update();
    camera_update(player_get_x(), player_get_y());
}
```
